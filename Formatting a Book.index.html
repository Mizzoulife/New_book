R Quarto: Formatting a Book

Quarto Book

quarto 파일들을 하나의 책으로 만들어 공유하기 위한 과정을 포함되어있다. 아래 과정을 따라하기 위한 예제 파일은 다운로드하여 연습해 볼 수 있다. (예제 다운로드)

Step 1. 파일 준비

Quarto html 문서를 책으로 묶기 위해서는 다음과 같은 파일들이 필요하다:

  1. Quarto.qmd
  2. Index.qmd
  3. _quarto.yml

책을 구성하기 위해서는 당연하게도 각각의 페이지들이 있어야 책으로 구성할 수 있다. 따라서, Quarto.qmd는 하나의 페이지로써 역할을 한다.

Index.qmd는 페이지와 별개로, 책을 구성하기 위한 필수 파일은 아니다. Index는 없어도 그만이지만 책의 소개 역할을 한다.

quarto.yml은 책을 구성하기 위한 필수 파일이다. quarto.yml은 R에서 따로 만들 수 있는 파일은 아니고, 직접 확장자명인 .yml을 넣어서 저장해야 한다. _quarto.yml은 책의 전체 설정 (책 제목, 목차, 포멧, 또는 테마)을 포함한다. 해당 파일의 포멧은 여기서 다운. (_quarto.yml 다운로드)

3개의 파일들은 한 폴더안에 유지하면 된다.

Step 2. Book 생성

파일이 준비 되었으니, R에서 실행해보자. .yml 확장자는 R에서 인식하기 힘들기 때문에 외부 확장자로 열릴 수 있다. R에서 편집하기 위해서는 .qmd 파일 중 아무거나 하나를 클릭하여 R로 오픈한 뒤, 우측 하단 Files에서 _quarto.yml을 클릭하면 R에서 열 수 있다.

R에서 모든 파일이 열린다면 _quarto.yml을 다음과 같이 추가하고자 하는 모든 .qmd를 목차에 알맞게 넣어주고, 글로벌 새팅을 완료해 주면 된다. 이때, _quarto.yml은 목록에 포함되지 않고, 같은 폴더에 위치하기만 하면 된다.

수정된 _quarto.yml을 저장하고, index.qmd 또는 아무 .qmd 파일에서 랜더 버튼()을 클릭하면 아래와 같이 _book 폴더가 자동으로 생성된다.

Step 3: Github에 Book 업로드

Github로 생성된 book을 공유하기 위해서는 _book폴더에 있는 모든 파일을 github에 업로드하면 된다.

깃허브에 로그인하고, Repositories로 들어가서 생성 버튼()을 누른다. 누른 다음 아래와 같은 화면이 나오고, 설정은 빨간 네모만 해주면 된다.

  • Repository name은 사용자 임의로 지정.
  • Choose visibility는 Public 또는 Privacy를 고를 수 있는데, 유료 회원이고 데이터를 보여주기 않으려면 privacy로 선택가능. 무료 회원은 Public만 가능.
  • Add README는 필수적으로 On.

생성이 완료되었고, 이제 파일 업로드 하면 된다. 해당 페이지에서 Add file를 클릭하여 Upload files를 선택.

Tip

한 번 업로드 되는 파일의 용량이 25MB가 넘지 않아야 함. 25MB가 넘는다면, Github앱을 컴퓨터에 설치하여 업로드 가능.

각 .html은 개별적으로 마우스를 끌어다 놓아도 가능하고, choose your files 버튼을 통해 업로드 해도 된다.

site_libs는 폴더 자체를 끌어다 놓으면 쉽게 업로드 가능. Commit changes ()클릭.

위와 같이 업로드가 완료 되었다면, 공유할 준비가 완료 되었다.

공유할 수 있는 인터넷 주소를 생성하기 위해, 빨간 네모의 Settings 클릭.
